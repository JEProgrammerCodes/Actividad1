I. Tema 1: Elementos de Páginas Web

I. Orígenes de la Web

    Web 2.0: Enfocada en la interacción y colaboración, permite a los usuarios generar y compartir contenido más fácilmente.
    Web 3.0:También conocida como la web semántica, busca mejorar la capacidad de las máquinas para entender y procesar la información de manera más inteligente.
    W3C:Consorcio que desarrolla estándares y directrices para garantizar la interoperabilidad y el desarrollo de la web.

II. Lenguajes de etiquetas

    SGML: Lenguaje de marcado para definir la estructura de documentos complejos.
    XML: Lenguaje de marcado flexible y extensible que permite crear etiquetas personalizadas para organizar datos.
    HTML: Lenguaje de marcado utilizado para construir y estructurar páginas web.
    XHTML:Variante de HTML que sigue las reglas de XML para una mayor precisión y rigor en la sintaxis.
    HTML5:La versión más reciente de HTML que introduce nuevas funcionalidades y mejoras para el desarrollo web, como soporte para multimedia y nuevas APIs.

III. Otros lenguajes relacionados al desarrollo web

    CSS/CSS3: Lenguaje para el estilo de páginas.
    JavaScript: Lenguaje para interactividad.
    Ajax: Técnica para actualizar contenido sin recargas.
    Tecnologías del lado del servidor: PHP, Python, etc.
    Navegadores: Herramientas para visualizar la web.
    Motores renderizados: Interpretadores de código.

IV. Herramientas de desarrollo

    Software WYSIWYG: Herramientas visuales para crear contenido.
    Otras alternativas: Figma, InVision, Zeplin, GitHub.
    Inspección de código: Herramientas en navegadores.

Estructura De Una Página Web

I. Jerarquía de directorio de un proyecto web

La jerarquía de directorio de un proyecto web se refiere a la forma en que se organizan los archivos y carpetas dentro de un proyecto. Generalmente se recomienda tener una estructura organizada que facilite el mantenimiento y la escalabilidad del proyecto. Un ejemplo de jerarquía básica podría incluir directorios como:

    index.html (Página de inicio)
    css/ (Carpeta para archivos CSS)
    js/ (Carpeta para archivos JavaScript)
    images/ (Carpeta para imágenes)
    assets/ (Carpeta para otros recursos como fuentes)
    pages/ (Carpeta para otras páginas HTML)

II. Elementos de estructura para el diseño de una página Web: Concepto

1. Regla utilizada para validar y certificar una página web

Para validar y certificar una página web, se utiliza la regla de validación del W3C (World Wide Web Consortium). Esta regla incluye el uso de validadores que analizan el código HTML y CSS para asegurar que cumplen con los estándares establecidos, como la validación de la sintaxis y la accesibilidad.

2. Reglas para escribir HTML

Algunas reglas básicas para escribir HTML incluyen:

    Usar etiquetas HTML apropiadas y cerrar todas las etiquetas que lo requieran.
    Utilizar la declaración DOCTYPE al comienzo del documento.
    Organizar el código y mantener una jerarquía clara de las etiquetas.
    Incluir metadatos importantes dentro de la etiqueta <head>.

    Usar atributos correctamente y evitar el uso innecesario de etiquetas obsoletas.

3. Estructura básica de un archivo web

La estructura básica de un archivo web en HTML es la siguiente: 

        <!DOCTYPE html>
        <html lang="es">
            <head>
                <meta charset="UTF-8">
                <meta name="viewport" content="width=device-width, initial-scale=1.0">
                <title>Título de la Página</title>
            </head>
            <body>
                <h1>Contenido Principal</h1>
                <p>Texto de ejemplo.</p>
            </body>
        </html>
